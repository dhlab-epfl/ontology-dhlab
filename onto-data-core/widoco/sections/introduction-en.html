Introduction back to ToC

The Core Data Model ontology describes the core elements which need to be represented when extracting information from texts. When dealing with textual data, a obvious aspect to model is the difference between what appears in the text (the word) and what it is being referred to (the ‘thing’, through the meaning). In the case of lexical information, this amounts to distinguish between an occurrence of a word in a text and its meaning as stated in a dictionary. When dealing with entities, this distinction corresponds to the mention(s) of an entity in texts on the one hand and, on the other, the entity of the world they refer to. As a consequence, the present ontology defines 3 main classes:

  • Source, i.e. an information object which mentions a
  • Mention, i.e. a fragment of the information object which refers to a
  • Entity, i.e. a real world entity of concept with distinct and independent existence.

These classes represent traditional and widely used textual annotation objects. As such, they compose the skeleton of various DHLAB activities involving text annotation, which is meant to be specialized according to projects' specific needs.

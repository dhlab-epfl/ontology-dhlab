Introduction back to ToC

The garzoni ontology describes the specific domain of interest of the archival document series Accordi dei Garzoni, kept at the State Archives of Venice.

This document series contains about 55.000 apprenticeship contracts declared by guilds of various professions over an almost continuous two-century period (1575-1772). The working arrangements contained therein represent an important source of information to better understand the driving forces of pre-modern economy, the transmission of skills, educational patterns and, in the case of Venice, artistic education. Extracted information — at an unprecedented scale in the domain of apprenticeship — will support enquiries such as who was working where, for how long and how much, how workshops were organized and distributed over the city, which professions were taught and where apprentices were coming from.

The Accordi dei Garzoni

Recorded over 32 registers by various different hands, apprenticeship contracts report information in an extremely regular manner (it was administrative documents). This is of particular benefit for the task of data modeling, and the present ontology naturally reflects the pieces of information present in the contracts, that is to say:
  • the identity of the involved persons, with the Apprentice, the Master, and potentially a Guarantor. Various information such as age, profession, residence or the geographical origins of persons are often mentioned. 
  •  the profession to be taught and the workshop where the learning takes place. 
  •  the diverse terms of the contracts, including: financial conditions (salary and pledge), hosting conditions (whether the master was offering the accommodation to the apprentice, or other advantages), and temporal conditions, with contract's starting date and duration.

Key modeling aspects

1.2.1 An integrated model

The garzoni ontology is not monolithic but builds on two ontologies and re-use other vocabularies.

The two ontologies (or vocabularies) named core data model ontology and common data model ontology are base vocabularies which are also used in similar projects dealing with archival material from the State Archive of Venice:

  • The Core Data Model Ontology consists of 3 main classes defining a Source (a document, a painting, a map) which mentions Mentions, which refer to Entities.

    For further information on this ontology please visit: http://vocab.dhlab.epfl.ch/data-core.

  • The Common Data Model Ontology builds on the Core Data Model and defines sub-classes (e.g. Person, Place) which are common to various projects.

    For further information on this ontology please visit: http://vocab.dhlab.epfl.ch/data-common.

The definition and organisation of these vocabularies is designed to be flexible and modular so as to ensure data integration from different but related projects.

Additionally, the representation of garzoni data makes use of other vocabularies:

  • the Web Annotation Data Model to represent provenance (cf. Section 1.2.2 below);
  • the Simple Event Model Ontology to represent event entity and their properties;
  • An archival metadata model, which temporarily defines a few elements, waiting for the validation of the Records in Contexts Conceptual Model;
  • the DHCanvas Annotation Model, that is to say some elements of the Annotation Model of the annotation tool with which the garzoni data is produced (cf. Section 1.2.2 below);
  • vocabulary elements from the IIIF presentation API (which integrates elements from the Shared Canvas Data Model).

1.2.2 The result of an annotation process

The garzoni ontology is used to represent data manually acquired via a dedicated transcription and annotation web interface, named DHCanvas. The internal annotation model of this application makes use of IIIF Image and Presentation APIs, and of the Web Annotation Data Model. In this frame, each annotated entity (or object) is the oa:Body of a oa:Annotation, which targets either image segments or other entities. Annotation are characterized with motivation and creation process information. In this context, the acquired data, encoded according to garzoni ontology, are naturally "wrapped" by annotations detailing their provenance. This information is not part, sensu stricto, of the Garzoni Data Model Ontology, but is an integral part of the garzoni data-set.

Introduction back to ToC

The Common Data Model Ontology does not target a specific domain of interest but defines the most common entities and relations we encounter when dealing with texts where strong referential units — a.k.a 'named entities' — are under focus. This vocabulary acts as an intermediate specification layer between the Core Data Model Ontology, and other more specialized vocabularies (such as the Garzoni Data Model). Other existing ontologies define similar entities (e.g. the DBPedia ontology, Schema.org), with which the present vocabulary will be linked.
